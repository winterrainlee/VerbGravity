# 버전 기획 (Plan) 워크플로우

다음 버전을 위한 기능 선정 및 기획을 수행합니다.

## 0단계: 기존 문서 참조

다음 문서들을 먼저 확인하여 현재 상태와 맥락을 파악합니다:
1. `docs/roadmap.md` - 전체 로드맵과 진행 상황
2. `docs/spec-v*.md` - 최신 버전 명세서 (가장 높은 버전)
3. `docs/logs/release/release-notes-v*.md` - 최근 릴리즈 노트

**확인 포인트:**
- 현재 버전에서 완료된 기능
- 로드맵에서 아직 구현되지 않은 항목
- 알려진 이슈나 TODO 항목

---

## 1단계: 프로젝트 분석 및 AI 제안

프로젝트 구조, 코드, 문서를 분석하여 다음 관점에서 개선점을 제안합니다:

### 분석 관점
1. **사용자 경험 (UX)**: 학생/교사의 사용 흐름에서 개선할 점
2. **기능 완성도**: 미완성 기능, 확장 가능한 기능
3. **기술 부채**: 리팩토링 필요 영역, 성능 개선
4. **버그/이슈**: 코드 리뷰 중 발견된 잠재적 문제

### 제안 형식
```markdown
### AI 제안 목록

#### 🚀 기능 개선
1. [제안 1] - 예상 작업량: 소/중/대
2. [제안 2] - 예상 작업량: 소/중/대

#### 🐛 버그 수정
1. [이슈 1] - 심각도: 낮음/중간/높음

#### 🔧 기술 개선
1. [리팩토링 1] - 우선순위: 낮음/중간/높음
```

---

## 2단계: 사용자 희망사항 질문

AI 제안 후, 사용자에게 다음을 질문합니다:

> **질문:**
> 1. 위 제안 중 이번 버전에 포함하고 싶은 항목이 있나요?
> 2. 별도로 추가하고 싶은 기능이나 개선 사항이 있나요?
> 3. 이번 버전의 목표 크기는 어느 정도인가요? (Minor / Patch)

---

## 3단계: 우선순위 결정

사용자 답변을 바탕으로 최종 기능 목록을 정리합니다:

### 정리 형식
```markdown
## 다음 버전 (vX.Y) 기능 목록

### 필수 (Must Have)
- [ ] ...

### 권장 (Should Have)
- [ ] ...

### 선택 (Nice to Have)
- [ ] ...
```

**확인 질문:**
> 위 목록으로 진행해도 될까요? 수정이 필요한 항목이 있다면 알려주세요.

---

## 4단계: 버전 명세 초안 생성

확정된 기능 목록을 바탕으로 버전 명세서 초안을 생성합니다:

1. **파일 생성**: `docs/spec-vX.Y.md`
2. **템플릿 사용**:
   ```markdown
   # 버전 명세서 (Specification) - vX.Y

   ## 개요
   - 목표: [버전의 핵심 목표]
   - 예상 기간: [작업 예상 기간]

   ## 기능 목록

   ### 1. [기능명]
   - 설명: ...
   - 영향 범위: Frontend / Backend / Both
   - 관련 파일: ...

   ### 2. [기능명]
   - ...

   ## 기술 요구사항
   - ...

   ## 검증 계획
   - ...
   ```

3. **사용자 리뷰 요청**:
   > 버전 명세서 초안을 생성했습니다. 내용을 확인하고 수정이 필요한 부분을 알려주세요.
