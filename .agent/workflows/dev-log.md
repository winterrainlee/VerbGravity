# 개발 일지 워크플로우

## 1. 버전 확인
- 현재 목표 버전을 확인합니다 (예: v1.1).

## 2. 파일 존재 여부 확인
- 경로: `docs/dev-log-v{버전}.md`
- 파일이 **존재하지 않으면** → 3단계 (새 파일 생성)
- 파일이 **존재하면** → 4단계 (기존 파일 갱신)

---

## 3. 새 파일 생성 (파일이 없는 경우)

**템플릿:**
```markdown
# 개발 일지 (Development Log) - v{버전}

## 상태 (Status)
- [ ] 기능 범위 정의 (Feature Scope)
- [ ] 프론트엔드 구현 (Frontend Implementation)
- [ ] 백엔드 구현 (Backend Implementation)
- [ ] 통합 및 테스트 (Integration & Testing)
- [ ] 배포 (Deployment)

## 일지 항목 (Log Entries)

### {날짜} - 버전 시작
- feature/v{버전} 브랜치 생성.
- 개발 일지 초기화.

## 트러블슈팅 (Troubleshooting)

| 문제 | 원인 | 해결 |
|------|------|------|
| - | - | - |

## 교훈 (Lessons Learned)
- (버전 완료 후 작성)
```

---

## 4. 기존 파일 갱신 (파일이 있는 경우)

### 4.1 일지 항목 추가
`## 일지 항목 (Log Entries)` 섹션의 **맨 아래**에 새 항목 추가:

```markdown
### {날짜} - {작업 제목}
- {작업 내용 1}
- {작업 내용 2}
```

### 4.2 상태 체크리스트 갱신
진행 상황에 따라 체크리스트 업데이트:
- `[ ]` → `[/]` (진행 중)
- `[/]` → `[x]` (완료)

---

## 5. 완료 알림
- 갱신된 내용을 사용자에게 요약하여 알립니다.
