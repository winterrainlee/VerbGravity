# 개발 로그 - v1.1.2 패치

## 📅 날짜: 2026-01-20

### 오늘 진행한 작업
1.  **주어구 힌트 로직 수정**: 기초 모드에서 주어 힌트 범위를 `Head`에서 `Span`으로 확대하여 사용자 직관성 개선.
2.  **시작화면 채점 모드 배지 추가**: 현재 설정된 모드(기초/심화)를 명확히 표시.
3.  **NLP 엔진 고도화**: 
    - `advcl`(부사절) 동사 인식 및 `ccomp` 지원 강화.
    - 관계절(`relcl`) 내 주어구 정밀 분리 로직 추가.
    - **모드별 차별화 원칙 적용**: 기초(단일 동사) / 심화(다수 동사) 분석 필터링 구현.
4.  **시스템 안정성**: NLP 분석 오류 발생 시 상세 로그 수집 체계(`nlp_errors.log`) 구축.
5.  **검증 및 배포**:
    - Pytest 5종 자동화 테스트 통과.
    - 부사절 문장을 이용한 기초/심화 모드 수동 검증 완료.
    - `master` 브랜치 병합 및 `v1.1.2` 태깅/푸시 완료.

### 배운 점 (Lessons Learned)
- spaCy의 의존성 구문 분석(`dep_`)을 활용할 때, `advcl`과 `ccomp`의 차이를 이해하고 이를 모드별 난이도 조절에 어떻게 반영할지 학습함.
- 프론트엔드와 백엔드 간의 `mode` 정보 동기화를 통해 분석 정밀도를 동적으로 조절하는 아키텍처를 실무에 적용해봄.
- 사용자 원칙("기초는 1개, 심화는 다수")을 기술적으로 구현하기 위해 서버 사이드 필터링이 유연한 해결책임을 확인.

### 다음 계획
- 수집된 `nlp_errors.log`를 정기적으로 검토하여 예외 케이스(도치문 등)에 대한 대응 로직 추가.
- 학생별 학습 데이터를 시각화하는 대시보드 고도화 (v1.2.0 준비).
